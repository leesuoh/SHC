2단계: 데이터베이스 연동 및 JPA 엔티티(Entity) 구현 (Database Setup)
로컬에 Docker를 이용해 PostgreSQL DB를 띄우거나, README에 명시한 Supabase 프로젝트를 생성하여 DB URL을 확보합니다.

Spring Boot 프로젝트에서 DB 연결 설정을 하고, README에 설계했던 4개의 핵심 테이블(Users, Presets, WorkOrders, WorkItems)을 JPA Entity 클래스로 작성합니다.

-------------------------------------------------------------------------------------------------------------------------------------------

3단계: 핵심 코어 API 개발 (Spring Boot API)
프론트엔드나 AI 연동 없이, 먼저 백엔드가 데이터를 잘 주고받는지 포스트맨(Postman) 등으로 테스트합니다.

인증/인가: 관리자와 정비사를 구분하기 위한 로그인 API (Spring Security + JWT) 구현.

리프트 상태 관리 API: 차량 입차, 리프트 할당, 정비 내역 추가, 핸드오버(인계), 출차 등의 핵심 CRUD API 개발.

-------------------------------------------------------------------------------------------------------------------------------------------

4단계: 모바일 프론트엔드 UI/UX 개발 (React)
백엔드 API가 어느 정도 준비되면, 정비사들이 현장에서 직접 터치할 화면을 만듭니다.

Tailwind CSS를 적용하여 화면 세팅.

4개의 리프트 상태가 한눈에 보이는 메인 대시보드 화면 퍼블리싱.

API 연동 (차량 번호 입력, 결제 승인 요청 등 로직 연결).

-------------------------------------------------------------------------------------------------------------------------------------------

5단계: 비전 AI 마이크로서비스 연동 (FastAPI + OpenAI)
핵심 비즈니스 로직(입차~출차)이 완성되면, 자동화의 핵심인 OCR 기능을 붙입니다.

FastAPI에서 이미지를 받아 OpenAI gpt-4o-mini 모델로 전송.

차량 번호판, 계기판(주행거리) 텍스트를 추출하여 Spring Boot 또는 프론트엔드로 전달하는 로직 구현.
